## The li numbering scheme (expanded 2026-06-25: 3-digit → 4-digit, milestone-grouped)

`li-MNNN` — **M** (leading digit) = the **milestone number**; **NNN** = the instruction within it.
- **`li-M000`** = the **milestone INDEX** (special): names what the milestone means + lists its
  constituent instructions. (This file = `li-1000`.)
- **`li-M001 … li-M999`** = the constituent L1i instructions for milestone M.
- **Filenames are number-only: `l1i/li-NNNN.md`** (no descriptive suffix). `l1i/li-X000.md` is
  *implicitly* the index for `li-X001 … li-X999`. (4 digits also read cleanly against 3-digit
  `op-NNN` / `id-NNN`.)

## What milestone 1 (1.0-preview) MEANS

**1.0-preview = the four core services {launchd, libnotify, asl, libxpc} all truly-green over the
{libdispatch, mach-ipc} substrate, with launchd hosting them, holding under integration soak.**

- **Expanded gate (Coordinator 2026-06-25):** the narrow Mach/dispatch/notify floor is a *waypoint
  inside* this gate, NOT a separate earlier ship point. All four core services gate the preview.
- **Get-it-right over speed:** "preview" is a *public label*, not license to ship thin. Target =
  **stable + usable**. This STRENGTHENS truly-green / overclaim-strict; it does not relax it.
- **North star — dogfood rmxOS:** the motivating goal is that we develop ON rmxOS, off the FreeBSD-15
  host — solid enough to live on day-to-day. (Boundary: full self-host buildworld-on-rmxOS + GPU stay
  the long arc / milestone 9; dogfooding is a quality north star, not yet a hard preview *gate* —
  Coordinator to confirm if it should be promoted to a gate.)
- **Image provenance:** ships on the proven base+overlay+pre-staged-kernel staging model, NOT the
  release pipeline (reproducible build = li-9xxx, post-preview). Cutting scope, not quality.
