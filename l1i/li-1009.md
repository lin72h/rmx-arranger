# li-1009 — x86-64-v3 platform baseline (base + ports + pkg; no legacy)

li-1009 | tier: L1i | milestone 1 (1.0-preview) constituent | status: ACTIVE (tryout) | owner: Coordinator 2026-06-25

WHAT IT MEANS: all of rmxOS — base system, ports tree, and binary pkg — compiled at **x86-64-v3**
(AVX2/BMI2/FMA, Haswell-2013+). Explicit decision: **drop legacy / pre-Haswell support.**
Mechanism (VERIFIED first-hand in `freebsd-src-official-stable-15`): `bsd.cpu.mk` recognizes the
`x86-64-v3` CPUTYPE token and maps `_CPUCFLAGS = -march=x86-64-v3` into CFLAGS/CXXFLAGS on amd64.
`/etc/make.conf` (and poudriere make.conf for ports/pkg):
```
CPUTYPE?=x86-64-v3
COPTFLAGS= -O2 -pipe
```
`COPTFLAGS=-O2 -pipe` is safe: `kern.pre.mk:67` auto-appends `-fno-strict-aliasing` for any `-O[23s]`, so
strict-aliasing protection is preserved even with `=`.

TRULY-GREEN BAR:
- base `buildworld`+`buildkernel` complete clean at v3; image boots clean.
- **kernel verified NOT emitting AVX/SSE in-kernel.** Note `kern.pre.mk:71` DOES append `_CPUCFLAGS`
  (`-march=x86-64-v3`) to the kernel's COPTFLAGS — but `kern.mk:133-134` appends `-mno-aes -mno-avx`
  (clang) + `-mno-mmx -mno-sse -msoft-float` AFTER it, so the `-mno-*` win and no vector codegen reaches
  the kernel (BMI2/GPR-only v3 features are harmless). Safe by construction — op-149 step 3 CONFIRMS this
  empirically (else FPU-state corruption/panic).
- v3 codegen confirmed actually present in userland (sanity that the flag took — e.g. AVX2 insn in a base binary).
- ports/pkg rebuilt at v3; resulting binaries run on a v3 host/guest.
- no-legacy stance documented (pre-Haswell unsupported).

LANDMINES: (a) kernel-AVX (above) — the one true correctness risk; (b) dev host + bhyve guest vCPU must
expose v3; (c) interacts with id-017 (buildworld skip-llvm-bootstrap) — bootstrap toolchain path.

PHASES: **P1 (now)** base buildworld+kernel v3 tryout — boot + kernel-AVX check + codegen sanity
(id-026 / op-149, queued overnight). **P2** ports/pkg via poudriere make.conf. **P3** default the staged
image to v3.
