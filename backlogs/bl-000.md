# bl-000 — Backlog overview + index

The **backlog** is the instruction stream *upstream of issue* — items identified but **not
yet fetched into the IDQ/ROB**. In the CPU/OoO model the rest of the workflow uses:

- **backlog (`bl-NNN`)** = instructions sitting in memory, not yet fetched. Planning-level.
  No pipeline, no agent assigned, no evidence spend.
- **op (`op-NNN`)** = an instruction that has been *fetched → decoded → issued*; now in
  flight in the IDQ/ROB (executing / waiting / retiring), owned by a pipeline + agent.

**Promotion = fetch.** When a `bl-NNN`'s preconditions are met and the Coordinator decides
to work it, it is *fetched*: decomposed into one or more `op-NNN` and issued. The backlog
item then points at the op(s) that carry it and is no longer planning-level.

`bl-NNN` is zero-padded, sequential, never reused (same discipline as `op-NNN`). `bl-000`
is this index; entries start at `bl-001`.

## State vocabulary (mirrors the op states)

| state | meaning (backlog) |
|---|---|
| **WAITING** | preconditions unmet — names what it waits on: a Coordinator decision, or an in-flight `op-NNN` that must retire first. Analogous to an op in the reservation station. |
| **READY** | preconditions met; fetchable now — ready to be decomposed into op(s) and issued. |
| **FETCHED → op-NNN** | promoted out of the backlog into flight; tracked thereafter by its op id(s) (the ROB owns it). Kept here with the pointer. |
| **RETIRED** | its op-chain completed. Terminal; kept for history. |
| **DROPPED** | abandoned before promotion (decision reversed / superseded). Kept for history with the reason. |

## Maintenance

Add a row when a `bl-NNN` is raised; update its state as it moves WAITING → READY →
FETCHED → RETIRED. When fetched, record the `op-NNN` it became in both this row and the
entry file. Keep this index terse — one row per item; detail lives in the entry file.
